MyCase KPI Reports — How to Pull & Structure

This guide covers how to extract, structure, and interpret monthly KPI reports from MyCase for operations reporting. Reports are pulled from MyCase, then organized in Google Sheets using pivot tables and formulas.

Platform: MyCase + Google Sheets  |  Audience: Operations team  |  Cadence: Monthly

1. Monthly Report Inventory

Each month, the following reports are generated:

Report NameWhat It ShowsSource
All Cases Still OpenFull case list of open cases at month-end, with how long each case has been in its current stageMyCase → Google Sheets
Time Entry (Monthly)Total time entries per user for the month — this is what "active cases" means: cases with at least one time entry that monthMyCase Firm Time Entry → Sheets
Active CasesNumber of distinct cases per user with time entries (unique case count, not hours)Pivot from Time Entry sheet
Opened CasesCases opened during the month, grouped by lead attorneyMyCase → Sheets
Closed CasesCases closed during the monthMyCase → Sheets
Overdue Task %Percentage of tasks that were incomplete and past due date, broken down by userMyCase Tasks → Sheets

2. Pulling Time Entry Data

To generate both the Time Entry and Active Cases reports:

  1. In MyCase, go to Firm Time Entry
  2. Select the date range for the month
  3. Enable the Detailed view option
  4. Export the report
  5. In Google Sheets, create a pivot table from the exported data

Time Entry Pivot (Total Hours per User)

Rows: User name  |  Values: SUM of Duration/Quantity

Example output:

User SUM of Duration/Quantity Ahmet Cemil Karakurt 194.5 Ahsen Akbulut 129.9 ...

Active Cases Pivot (Distinct Cases per User)

Using the same time entry data, create a second pivot:

Rows: User name  |  Values: COUNTUNIQUE of Case Link

This gives the number of distinct cases each user logged time on — this is the "active case count."

3. All Cases Still Open — Stage Duration Extraction

MyCase shows stage duration in a format like AS - I-589 Filed (670 days). To extract the day count cleanly:

  1. Use a formula to separate the stage name from the parenthetical number
  2. Extract just the numeric value (670 in the example above)
  3. Use this to calculate averages across cases in the same stage

The final report contains two tabs: a full case list with all details, and an average-days-per-stage summary.

Key Formulas Used

FormulaPurpose
=UNIQUE(FLATTEN(Sheet2!A:H))Get all unique assignee names across multi-person task assignments
=ARRAYFORMULA(COUNTIF(Sheet2!A:H, A1:A))Count how many times each name appears across the task table
=TEXT(K24/86400, "mm:ss")Convert seconds to MM:SS duration format
=ARRAYFORMULA("00:" & AA:AA)Prefix a "00:" to a minutes column to build HH:MM format

4. Opened & Closed Cases

These reports show cases by the month they were opened or closed. In addition to a count-per-attorney pivot, a detailed list is generated showing the attorney name followed by each case they opened/closed.

Example structure for Opened Cases:

Row Labels Count of Column1 Ahsen Akbulut 1 ↳ Yifeng Shi - Marriage-Based Green Card Ali Aricak 3 ↳ Ayhan Karaduman - E2 ↳ Kivanc Senyurek - E-2 CoS ↳ Mustafa Calisir - E2

Note: Rows without a lead attorney assigned appear as blank entries — these represent cases where lead attorney selection was skipped at intake.

5. Overdue Task Percentage

To calculate the overdue task rate per user:

  1. From the MyCase Tasks section, pull:
    • All users — selected month — Incomplete only → this gives overdue/incomplete tasks
    • All users — selected month — All tasks → this gives the total task count
  2. Filter both exports so only tasks with a due date within the target month are included — MyCase's date filter may not always match exactly, so review the due date column
  3. Break down by attorney/user
  4. Use the FLATTEN + COUNTIF formulas to handle tasks that have multiple names assigned
⚠ Important: A single task can have multiple assignees. If you count rows directly, each name gets counted once per row even if they share it. Use the FLATTEN approach to count per-person correctly.

6. Quote-to-Case Workflow — Automation Error Handling

When a deal is marked as closed-won and a signed quote triggers case creation in MyCase via Zapier, the following errors can occur:

ErrorCauseResolution
Lookup value missing — Google SheetsThe deal owner's email is not in Sheet 3 of the user ID mapping sheet, OR the user's email changed (e.g. Rippling deletion/re-add)Open the Zapier run → get deal data → find deal owner email → add to Sheet 3 with format: id (required) — no value — name — surname — email (required). If the owner is in Intake, also add them to the Deal Owner Copy property in the Deal Object and add the relevant branch to the Zapier workflow in HubSpot.
Name has already been takenA case with the same name already exists in MyCaseFind the existing case, determine if it's a duplicate. If so, rename one of them (e.g. add a dash suffix). Then replay the Zapier run.
The user specified is duplicateThe deal owner is already assigned as a required staff member in the Create Case action (e.g. a director who normally doesn't do sales closed a deal)Remove the duplicate mapping in the automation's Create Case step. Follow the video guide in the SOP Library folder.

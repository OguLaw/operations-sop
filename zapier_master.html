⚠️ Critical: Sub-Zaps Must Always Stay ON

Sub-zaps (#7, #16, #17, #19, #20) must never be turned off. They have no external trigger — they're called by parent zaps. If a sub-zap is off, the parent zap will fail silently at that step. Before turning anything off for maintenance, check whether it's called by another zap.

Zap #9 — deal kazanılınca mycasete case + contact ⭐ Critical

The most critical automation in the workspace. When a deal is marked Closed Won in HubSpot, it automatically opens a case in MyCase, assigns it to the correct attorney, sends notification emails, and updates the deal record.

Step-by-Step Flow

  1. Trigger — HubSpot: deal property change → Is Closed Won
  2. Filter — Confirm deal is actually closed won
  3. Get Deal — Retrieve full deal details (case type, pipeline, quote info, contact ID)
  4. Get Contact — Retrieve contact details (formatted phone, case types)
  5. Filter — Check contact is valid
  6. Update Contact — Tag contact in HubSpot
  7. MyCase Search — Search for existing person in MyCase
  8. Sheets Lookup — Look up staff user IDs from user ids / Sheet3
  9. Branch: Person found? — Path "buldu" (found) or "bulamadı" (not found)
  10. Format Date — Format retainer/case start date
  11. Branch: Attorney — Nur Kaldırım / Yusuf / Gulbabil
  12. Per attorney path: Create MyCase case → Send ops email → Send client email → Update HubSpot deal → Call Sub-Zap #20
  13. Branch: Case type (for new person path) — CoS / Family / Business Immigration
  14. Create Person — MyCase: create new person if not found, then repeat case creation flow

➕ How to Add a New Attorney

  1. Open the Zap → find the Branch: Attorney step
  2. Add a new path with the filter condition matching the attorney's HubSpot owner ID or name
  3. Duplicate an existing attorney's block (caseCreate → ops email → client email → update deal → call sub-zap)
  4. Update the staff field in the caseCreate step with the new attorney's MyCase staff name
  5. Test with a dummy closed-won deal

Zap #24 — Referral Company Signed Deal Email (Weekly)

Every Tuesday at 6PM, checks the Referral Notification sheet for rows with status "pending", groups them by referral company, generates an HTML email table of signed clients, emails each company from information@oguz.law, and marks rows as "sent".

🔧 Critical Maintenance Note

The zap marks rows as "sent" only on the last loop iteration. If the last company has no clients (rowCount = 0), its send step is filtered out — and the entire sheet never gets marked as sent. Ensure the last company in the config tab always has at least one pending client, or restructure the update step to run outside the loop.

➕ How to Add a New Referral Company

Add a new row to the config tab in the Referral Notification sheet: Company name, Email, Official Name. Ensure incoming referral data in Sheet1 uses exactly the same company name (case-insensitive match is used in the JS code). No Zapier changes needed — the loop auto-iterates.

Zaps #25–35 — Feedback Tracker System (9 Zaps)

A 9-zap system that routes internal feedback from a Slack channel to department Trello boards, then notifies the original submitter when their feedback is resolved or rejected.

Zap #25 — Slack → Trello Routing

Reads structured messages posted to #feedback-log Slack channel, parses department/name/subject/due date via regex, looks up Trello member and label IDs via lookup tables, and creates a card on the correct department board.

Message format required: fields for Department, Name, Subject, Response Needed, Reply Date — the Formatter steps use regex Extract Pattern to pull each field. If the Slack message format changes, all 6 Formatter steps must be updated.

Zaps #26–32 — Completion Notifications

When a card is moved to "Completed" on a department board, each zap finds the original Slack message (by card title), extracts the submitter's name, looks up their Slack user, and DMs them in #feedback-log. One zap per board: Billing, HR, Marketing, Operasyon, Legal, Sales, Intake.

Zap #35 — Rejection Notification

When a card is archived on any board, finds the original Slack message, extracts both the submitter name and department head name, and notifies both via Slack.

➕ How to Add a New Department

  1. In Zap #25: add a new path to the Branch step, pointing to the new Trello board and list. Add department → Trello label ID mapping to the Lookup Table step. Add person → Trello member ID if needed.
  2. For completion notifications: duplicate any of Zaps #26–32, change the trigger to the new Trello board → Completed list, turn the new zap ON.

Zap #36 — (Tekli) Weekly Phone Number Processing and HubSpot Update Workflow

LSA (Local Services Ads) leadleri manuel olarak "LSA Leads" spreadsheet'ine (leads-girildi-4 worksheet'ine) girildikten sonra, kontrol sheet'indeki (Lsa için güncellemeli → Sheet1) güncel mi kolonu güncellendiğinde tetiklenir. Zap tüm lead listesini okur, 3 katmanlı dedupe kontrolünden geçirir ve her yeni numara için HubSpot'ta contact arar — varsa LSA lead-source alanlarıyla günceller, yoksa yeni bir contact ("LSA Lead Zapier") oluşturur. Sonsuz çalışacak şekilde kurgulanmıştır; manuel müdahale yalnızca tetiklemek veya bozulmaları düzeltmek için gerekir.

Step-by-Step Flow

  1. Trigger — Lsa için güncellemeli / Sheet1 sheet'inde bir satır güncellenir (güncel mi kolonu bazında dedupe)
  2. Get Many Rows — LSA Leads / leads-girildi-4 worksheet'inden tüm satırlar çekilir (500 row limit)
  3. Format Phone Number — tüm telefonlar US formatına dönüştürülür
  4. Line Itemizer — telefonlar loop'a hazır liste haline getirilir
  5. Loop — her telefon için aşağıdaki adımlar ayrı ayrı çalışır:
    1. Lookup in processed-sınır → bulunursa stop (daha önce işlenmiş)
    2. Lookup in processed-sınır2 → bulunursa stop
    3. Lookup in processed-sınır3 → bulunursa stop
    4. Delay — rate limit koruması
    5. Format Phone (tekrar, tek telefon bazında)
    6. HubSpot Search — contact'ı telefon numarasıyla ara
    7. Branch — HubSpot'ta contact var mı yok mu?
      • Path A (existing contact): update_crm_contact → LSA lead source alanlarını yaz → 3 dedupe sheet'i tekrar kontrol et → processed-sınır2'ye telefonu ekle
      • Path B (no contact): contactCreate (firstname: LSA Lead, lastname: Zapier, phone, LSA source alanları) → Created contacts worksheet'ine log at → 3 dedupe sheet'i tekrar kontrol et → processed-sınır2'ye telefonu ekle
Neden 3 katman dedupe? processed-sınır, processed-sınır2 ve processed-sınır3 farklı dönemlerde/kaynaklarda işlenmiş telefonları tutar. Loop her telefon için her üçünü sırayla kontrol eder; birinde bulunursa işleme alınmaz. Yeni işlenenlerin hepsi processed-sınır2'ye yazılır (bu "aktif" işlenmiş listesidir).

➕ Nasıl Yeni Lead Eklenir (Rutin Kullanım)

  1. LSA Leads spreadsheet'ini aç → leads-girildi-4 worksheet'ine git
  2. Yeni telefon numarasını A kolonuna (Customer) ekle — format önemli değil, Formatter normalize eder
  3. Lsa için güncellemeli spreadsheet'ine git → Sheet1 → güncel mi (B) kolonundaki herhangi bir hücreyi değiştir (veya yeni bir satıra x yaz)
  4. 15 dakika içinde Zapier trigger'ı yakalayıp tüm listeyi işleyecek — daha önce dedupe sheet'lerinde olanları atlayıp sadece yeni eklenen numaraları HubSpot'a yazacak

🔄 Bir Telefonu Yeniden İşletmek (Force Re-process)

  1. LSA Leads spreadsheet'ini aç
  2. processed-sınır, processed-sınır2, processed-sınır3 worksheet'lerinin üçünde de o telefonun satırını sil (sadece birinden silmek yetmez; loop üç sheet'i de kontrol ediyor)
  3. Lsa için güncellemeli / Sheet1 → güncel mi kolonunda bir hücre güncelle, zap yeniden çalışsın
